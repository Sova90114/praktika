Данный репозиторий содержит конфигурационные файлы, скрипты и документацию для развертывания автоматизированной инфраструктуры проекта. Проект охватывает полный цикл доставки ПО: от контейнеризации и CI/CD до оркестрации в Kubernetes, мониторинга и управления конфигурациями через IaC.

🌿 Структура репозитория и ветвление
В данном проекте принято следующее разделение веток:

master: Основная ветка разработки. Здесь хранятся все конфигурационные файлы, манифесты, плейбуки и скрипты.
main: Ветка документации. Содержит актуальную версию README.md и отчетные материалы.

⚠️ Примечание: При клонировании проекта для развертывания инфраструктуры используйте ветку master.

🐳 CI/CD и Контейнеризация
В этой секции реализована автоматизация сборки и доставки приложений.

GitLab Runner: Настроен и зарегистрирован раннер для выполнения задач пайплайна.
Dockerfile: Описаны инструкции для сборки образов приложений.
Nginx: Конфигурация веб-сервера для маршрутизации трафика.
GitLab CI: Настроен пайплайн (.gitlab-ci.yml) для автоматической сборки образов и деплоя.

☸️ Оркестрация (Kubernetes)
Инфраструктура контейнеризированных приложений управляется через кластер Kubernetes.

Кластер: Развернут на виртуальных машинах с помощью kubeadm согласно официальной документации.
Manifests: Деплой приложений реализован через нативные Kubernetes манифесты (Deployment, Service, Ingress и др.).
Helm: Альтернативный способ деплоя упакован в Helm-чарты для упрощения управления релизами.

📊 Мониторинг и Алертинг
Обеспечена наблюдаемость за состоянием инфраструктуры и контейнеров.

Стек мониторинга: Prometheus и Grafana развернуты через docker-compose.
Экспортеры:

docker-exporter: для сбора метрик контейнеров.
node-exporter: для сбора метрик хостов (развертывается через Ansible).

Алертинг: Настроены правила оповещения и дашборды в Grafana (в том числе для docker-executor).

🤖 Infrastructure as Code (Ansible)
Управление конфигурацией серверов автоматизировано с помощью Ansible.

Playbooks: Написаны плейбуки для развертывания компонентов (например, node-exporter).
CD для Ansible: Настроен процесс запуска плейбуков через GitLab CI.
Безопасность: Для хранения чувствительных данных (пароли, ключи) используется Ansible Vault.
