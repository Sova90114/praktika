Проект охватывает полный жизненный цикл доставки программного обеспечения: от контейнеризации и CI/CD до оркестрации в Kubernetes, мониторинга и управления конфигурациями через подход Infrastructure as Code (IaC).

CI/CD и контейнеризация

В данном разделе реализована автоматизация сборки и доставки приложений:

Dockerfile — описаны инструкции для сборки образов приложений.

Nginx — конфигурация веб-сервера для маршрутизации трафика.

GitLab CI — настроен пайплайн (.gitlab-ci.yml) для автоматической сборки образов и деплоя.

Оркестрация (Kubernetes)

Управление инфраструктурой контейнеризированных приложений осуществляется через кластер Kubernetes:

Кластер — развернут на виртуальных машинах с помощью kubeadm в соответствии с официальной документацией.

Манифесты — деплой приложений реализован через нативные Kubernetes-манифесты (Deployment, Service, Ingress и др.).

Helm — альтернативный способ деплоя, упакованный в Helm-чарты для упрощения управления релизами.

Мониторинг и алертинг

Обеспечена наблюдаемость состояния инфраструктуры и контейнеров:

Стек мониторинга — Prometheus и Grafana развернуты через docker-compose.

Экспортеры:

docker-exporter — для сбора метрик контейнеров;

node-exporter — для сбора метрик хостов (развертывается через Ansible).

Алертинг — настроены правила оповещения и дашборды в Grafana (включая дашборды для docker-exporter).

Infrastructure as Code (Ansible)

Управление конфигурацией серверов автоматизировано с помощью Ansible:

Playbooks — написаны плейбуки для развертывания компонентов (например, node-exporter).

CD для Ansible — настроен процесс запуска плейбуков через GitLab CI.
